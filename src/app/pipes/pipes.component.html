Chaining Pipes


Chaining of Pipes is nothing but adding more than one pipe and transforming the data as desired. Here the output data from one pipe is given as input to the next Pipe. But be cautious as chaining the pipes to complex data might be making the performance slow.


{{ user + user1 | slice : 0 : 13 }} - slicing from start to 13

{{ user + user1 | slice : 0 : 13 | slice : 4 : 10 }} - slicing from start to 13 and then from 4 to 10

{{ user + user1 | slice : 0 : 13 | slice : 4 : 10 | uppercase }} - slicing till 13 and then slicing from 4 to 10 and then to upperCase



Custom Pipes


You can define a custom pipe by implementing a TypeScript class with the @Pipe decorator. A pipe must have two things: A name, specified in the pipe decorator A method named transform that performs the value transformation. The TypeScript class should additionally implement the PipeTransform interface to ensure that it satisfies the type signature for a pipe.
